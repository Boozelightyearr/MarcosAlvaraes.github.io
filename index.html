Bem-vindo ao Meu Site

A acessibilidade web ajuda pessoas com deficiência a navegarem por um site e acederem conteúdos através de ferramentas específicas, como um leitor de ecrã. Ao tornar o site acessível, permitimos que mais pessoas possam interagir com o mesmo proporcionando uma boa experiência.

Projetos

Projeto 1

Pretende-se então construir um site pessoal com uma página, ou como também é denominado “one page”, acessível, que cumpra os aspetos de acessibilidade.

As Diretrizes de Acessibilidade para Conteúdo Web (WCAG) oferecem um padrão global de conteúdos acessíveis que cumprem com as obrigações legais e possibilitam a inclusão de todos assim como de organizações na internet.

Devemos tomar como nota de partida a diretrizes WCAG 2.0 (https://www.w3.org/Translations/WCAG20-pt-PT/). Esta diretrizes seguem 4 princípios: percetível, operável, compreensível e robusto.

Existem 3 níveis de conformidade, nível A, nível AA e nível AAA.

Para o nosso trabalho vamos apontar a elaboração para o nível AA.

Para uma conformidade de Nível AA, a página Web satisfaz todos os Critérios de Sucesso de Nível A e Nível AA ou é fornecida uma versão alternativa em conformidade com o Nível AA.

Projeto 2

1. Uso de HTML Semântico O conteúdo deve ser estruturado com elementos ou tags semânticas. Isto ajuda os leitores de ecrã a compreender a hierarquia do site.

2. Descrições Alternativas em Imagens As imagens devem ter texto descritivo, no seu atributo 'alt'. Por exemplo, em vez de "imagem1.jpg", utilize algo como "Fotografia de uma praia ao pôr do sol".

3. Contraste de Cores O texto deve ter contraste suficiente em relação ao fundo. Um bom contraste facilita a leitura para pessoas com baixa visão. Ferramentas como Contrast Checker são úteis para este fim.

4. Navegação via Teclado É importante que todos os elementos interativos (botões, links, menus) possam ser acedidos apenas com o teclado, utilizando para isso a tecla Tab.

5. Legendas e Transcrições Para vídeos e conteúdos multimédia, é importante o uso legendas e transcrições. Isto beneficia não apenas pessoas com deficiência auditiva, mas também utilizadores que preferem ler ou estão em ambientes ruidosos.

6. Feedback Visual e Auditivo Ao interagir com o site (como enviar um formulário), indicações claras, como mensagens de erro visíveis ou confirmações de ação bem-sucedida, são cruciais para o utilizador obter feedback da sua interação.

7. Evitar Depender Apenas de Cores Não se deve usar apenas cores para transmitir informações. Deve-se, por exemplo, combinar cor com ícones ou texto para reforçar a mensagem que se quer transmitir.
